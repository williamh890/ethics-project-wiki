    Home-made CSS

    This page is pretty ugly because it is CSS that I made. I just want to show you that it is possible to make your own, too. The menu is from the Dynamic Drive CSS Library.

    Be sure to use a unique name for your classes and id's, or it will conflict with the project management (PM) server CSS. To be frank, you will probably face this problem of "dueling CSS's," but just try to work around it. This should only be a minor inconvenience if you use unique names for your own CSS.

    You may find it interesting that I used an ordered list to create this 3-column main section. Try resizing your window height and width to see how well this works. You will find the CSS for this in group-projects.css.

    @media print

    Please add CSS to print your work. I created a simple @media print in group-projects.css as an example.

    Other CSS Options

    In the Wiki Editor

    You only need to place the code I wrote in the footer into your wiki main page for this method to work, but write your own project name instead of cs371_group_projects.

    Notice that each page does not have a unique URL, which affects search engine optimization (SEO). I am not worried about this because our projects are only for practice and group assignment grading, but feel free to make separate wiki pages if you want your pages to be directly navigable.

    Using Git

    In Netbeans, if you look under the Team menu, you will find all of the Git controls. After setting it up, you pretty much only need:

    Team

    • Add
    • Commit
    • Remote
      • Fetch
      • Pull
      • Push

    First, create a generic layout and push it to your repository because it may look slightly different on the PM server than on your local system. After that, you only need to push it to the PM server when it looks good on your local system, not after every change you make.

    Lastly, Netbeans seems to only commit files that you have selected, so be sure to select the parent folder, such as "Site Root" or the project folder, when committing multiple files.
